# QPaperMaker вҖ” Design Document

## Problem Statement

Teachers spend significant time manually writing question papers aligned to textbook content. This app automates that process: teachers upload photos of textbook pages, configure paper parameters, and receive a formatted PDF question paper generated by an AI pipeline.

## Agentic Framework: CrewAI

**Chosen over OpenAI Agents SDK** because:
- 4 distinct role-based agents map directly to CrewAI's design pattern
- Sequential task chaining with automatic context passing between agents
- No manual orchestration boilerplate
- Lightweight: all LLM calls go to OpenAI API (no local model inference)

## Data Flows

### Upload & Ingest

```
Teacher uploads images (1 or more)
  вҶ’ POST /api/books/upload
      Node.js: multer saves to /data/uploads/{book_id}/
      Node.js: inserts book row in SQLite (status = 'processing')
      Node.js: calls POST /ai/ocr + POST /ai/ingest asynchronously

  вҶ’ POST /ai/ocr  (FastAPI)
      For each image: GPT-4o Vision API вҶ’ raw text string
      Returns: { pages: [{page: 1, text: "..."}, ...] }

  вҶ’ POST /ai/ingest  (FastAPI)
      Chunk text (paragraph-level, ~500 tokens, 50-token overlap)
      OpenAI text-embedding-3-small вҶ’ vector per chunk
      Store in ChromaDB collection: "book_{book_id}"
      Returns: { chunks_stored: N }

  вҶ’ Node.js: updates book status to 'ready'
  вҶ’ React: polls GET /api/books/{id}/status until 'ready'
```

### Generate Paper

```
Teacher configures parameters вҶ’ POST /api/papers/generate
  вҶ’ Node.js: validates params, inserts paper row (status = 'generating')
  вҶ’ Node.js: calls POST /ai/generate

  вҶ’ POST /ai/generate  (FastAPI)
      1. RAG retrieval:
         - Query ChromaDB "book_{book_id}" with topic keywords
         - Retrieve top-20 most relevant chunks
      2. CrewAI sequential run:
         ContentAnalyst вҶ’ QuestionDesigner вҶ’ QualityReviewer вҶ’ PaperFormatter
      3. PDF generation:
         - PaperFormatter output JSON вҶ’ ReportLab вҶ’ /data/papers/{paper_id}.pdf
         - If include_answer_key: generate second PDF /data/papers/{paper_id}_key.pdf
      Returns: { paper_id, file_path }

  вҶ’ Node.js: updates paper record (status = 'ready', file_path)
  вҶ’ React: download link becomes active
```

## Database Schema (SQLite)

```sql
CREATE TABLE users (
  id           TEXT PRIMARY KEY,       -- uuid
  email        TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  created_at   DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE books (
  id           TEXT PRIMARY KEY,       -- uuid
  user_id      TEXT NOT NULL REFERENCES users(id),
  title        TEXT NOT NULL,
  page_count   INTEGER DEFAULT 0,
  status       TEXT DEFAULT 'processing', -- processing | ready | error
  error_msg    TEXT,
  created_at   DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE papers (
  id           TEXT PRIMARY KEY,       -- uuid
  user_id      TEXT NOT NULL REFERENCES users(id),
  book_id      TEXT NOT NULL REFERENCES books(id),
  title        TEXT NOT NULL,
  params_json  TEXT NOT NULL,          -- JSON blob of generation parameters
  status       TEXT DEFAULT 'generating', -- generating | ready | error
  file_path    TEXT,                   -- absolute path to PDF
  key_file_path TEXT,                  -- answer key PDF path (nullable)
  error_msg    TEXT,
  created_at   DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## REST API Contracts

### Auth (Node.js)

| Method | Path | Body | Response |
|---|---|---|---|
| POST | `/api/auth/register` | `{email, password}` | `{token, user}` |
| POST | `/api/auth/login` | `{email, password}` | `{token, user}` |

### Books (Node.js, JWT required)

| Method | Path | Notes |
|---|---|---|
| GET | `/api/books` | List user's books |
| POST | `/api/books/upload` | `multipart/form-data` with `title` + `images[]` |
| GET | `/api/books/:id/status` | Poll for `{status, page_count}` |
| DELETE | `/api/books/:id` | Deletes book + ChromaDB collection |

### Papers (Node.js, JWT required)

| Method | Path | Notes |
|---|---|---|
| GET | `/api/papers` | List user's papers |
| POST | `/api/papers/generate` | `{book_id, ...params}` вҶ’ starts async generation |
| GET | `/api/papers/:id/status` | Poll for `{status}` |
| GET | `/api/papers/:id/download` | Streams PDF file |
| GET | `/api/papers/:id/download-key` | Streams answer key PDF |

### AI Pipeline (Python, internal only)

| Method | Path | Notes |
|---|---|---|
| POST | `/ai/ocr` | `{images: [base64...]}` вҶ’ `{pages: [{page, text}]}` |
| POST | `/ai/ingest` | `{book_id, pages: [{page, text}]}` вҶ’ `{chunks_stored}` |

| POST | `/ai/generate` | `{book_id, params}` вҶ’ `{paper_id, file_path}` |
| GET | `/ai/health` | `{status: "ok"}` |

## HuggingFace Deployment

- **Space type**: Docker SDK
- **Port**: 7860 (HF requirement вҖ” Node.js binds here)
- **Persistent volume**: `/data` (set `DATA_DIR=/data` in HF secrets)
- **Secrets**: `OPENAI_API_KEY`, `JWT_SECRET`, `DATA_DIR`
- **supervisord**: starts `uvicorn` on :8000 + `node` on :7860
- **Cold start**: ~30вҖ“60s on free tier CPU; show loading screen in React
- **Sleep**: Free tier sleeps after inactivity вҖ” warn users in UI

### Dockerfile Strategy

```
Stage 1: node:20-alpine  вҶ’ npm ci + vite build вҶ’ /app/frontend/dist
Stage 2: python:3.11-slim вҶ’ pip install requirements
Stage 3: node:20-alpine (final)
  - Copy Node.js backend
  - Copy React dist
  - Copy Python venv from Stage 2
  - Copy ai_pipeline source
  - Install supervisord
  - EXPOSE 7860
  - CMD ["supervisord", "-c", "/app/supervisord.conf"]
```

## Tech Stack Rationale

| Choice | Why |
|---|---|
| CrewAI (not OpenAI Agents SDK) | 4-role sequential pipeline maps directly; less boilerplate |
| ChromaDB local mode | File-based, zero ops, works on HF free tier |
| SQLite + better-sqlite3 | File-based, synchronous API fits Express well |
| ReportLab (not WeasyPrint) | Pure Python, no system deps (WeasyPrint needs libpango) |
| GPT-4o Vision for OCR | Handles handwriting, diagrams, mixed layouts better than tesseract |
| text-embedding-3-small | 1536-dim, cheap ($0.02/1M tokens), good retrieval quality |
| supervisord | Lightweight multi-process manager, standard in single-container deploys |
